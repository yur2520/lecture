웹사이트의 구성 요소

웹사이트를 구성하는 코드는 언뜻 복잡한 글자와 문자들이 뒤얽혀 있는 것 같아 보입니다.

하지만 각 코드는 정해진 문법에 따라 역할이 나뉘어 있습니다. 건축물의 구성 요소와도 비슷합니다.

HTML: 구조 설계

HTML (HyperText Markup Language)은 건축물의 뼈대에 해당합니다.

예를 들자면, "이 부분에 제목이 들어가고, 그 아래 이 부분에 이미지를 넣을 거야."라는 식으로 틀을 잡아주는 것이 HTML의 역할입니다. 웹페이지의 기본 구조와 콘텐츠가 어떻게 이루어지는지 정의해 주는 코드라고 할 수 있습니다.

CSS: 시각 디자인

CSS (Cascading Style Sheets)는 건축물의 인테리어 디자인과 같습니다.

html코드로 "여기에 제목이 들어간다" 라는 사실을 지정했다고 가정해 보세요. 그 제목을 이루는 글자의 크기는 어느 정도인지, 글자의 색상은 무슨 색을 쓸 건지, 배치는 어떻게 할 건지와 같은 세부사항을 정해 주는 것이 바로 CSS 코드입니다. 웹사이트의 전반적인 시각적 표현을 담당하고 있는 코드예요.

JavaScript: 기능 설비

자바스크립트(JavaScript)는 건축물의 전기, 수도와 같은 기능 설비에 해당합니다. 웹 사이트의 특정 부분을 클릭했을 때 어떤 변화가 나타나게 하고 싶다고 가정해 봅시다. 예를 들어 팝업창을 띄운다거나 하는 일 말이에요. 이 때 사용자의 행동에 반응하게 해 주는 코드가 자바스크립트 코드입니다. 이 버튼을 클릭하면 이런 팝업창을 띄워라, 이런 식으로요. 즉, 웹사이트에서 동적인 상호작용을 가능하게 해 주는 코드라고 할 수 있습니다.

효율적인 도구의 등장과 웹사이트 제작 방식의 변화

Tailwind CSS

CSS가 인테리어 디자인이라면, Tailwind CSS는 '빌트인 가구'와 비슷합니다. 목재를 가져와 직접 설계하는 대신, 업체에서 제공하는 색상, 크기 등의 선택지들을 조합하여 가구를 완성하는 방식입니다.

JavaScript 라이브러리

자바스크립트 라이브러리는 건축물의 '스마트 스위치'와 같습니다. 복잡한 전기 회로를 직접 설계하는 대신, 미리 정교하게 만들어진 제품을 설치해 고급 기능을 바로 사용하는 것과 같습니다. '이미지 슬라이드' 같은 복잡한 기능을 예로 들 수 있습니다.

더 간편해진 웹페이지 만들기

전통적으로는 역할에 따라 HTML(구조), CSS(디자인), JavaScript(기능) 파일을 분리하는 것이 기본이었습니다. 하지만 이제는 간단한 웹사이트의 경우, HTML 파일 하나만으로도 충분히 만들 수 있습니다.

HTML 문서에 Tailwind CSS 같은 디자인 도구와 자바스크립트 라이브러리(기능)의 온라인 주소(CDN)를 연결하기만 하면, 별도의 파일을 만들지 않고도 빠르고 간편하게 하나의 완성된 웹페이지를 제작할 수 있게 되었습니다.

프론트엔드와 백엔드

웹 기술의 역할은 크게 두 가지로 나누어져요.

프론트엔드 (Frontend)

프론트엔드는 카페에서 테이블과 탁자가 있는 곳에 비유할 수 있습니다. 손님(사용자)이 직접 마주하고 경험하는 모든 시각적 요소와 상호작용이 바로 프론트엔드 부분입니다. 웹 환경에서는 사용자의 화면에 표시되는 모든 것을 의미합니다.

백엔드 (Backend)

백엔드는 카페 카운터 너머, 음료나 음식을 만들고 준비하는 곳으로 비유할 수 있습니다. 손님들이 직접 경험하는 부분은 아니지만, 보이지 않는 곳에서 누군가 주문을 처리하고, 재료를 가져와 준비하고, 남은 재료는 다시 창고에 넣는 등 바삐 움직이고 있는 것과 비슷해요. 백엔드는 웹사이트가 원활하게 작동하도록 지원하는 기술 분야에 해당하며, 데이터를 저장하거나 꺼내고, 관리하고, 처리(계산)하는 역할을 합니다.

웹사이트가 세상에 공개되는 과정

로컬과 네트워크, 서버와 호스팅 개념 이해하기

웹사이트를 요리라고 가정해 봅시다. 웹사이트를 만들고 세상에 공개하는 과정은 '우리 집 주방에서 요리하다가 전문 식당을 여는 과정'과 비슷해요.

  1. 1

    로컬 (Local): 우리 집 주방

    개발자의 개인 컴퓨터는 비공개적인 '우리 집 주방'과 같습니다. 이곳에서 웹사이트라는 요리를 개발하고 테스트합니다. 가족에게 요리를 대접할 수는 있지만, 24시간 불특정 다수의 손님을 받기에는 적합하지 않습니다.

  2. 2

    서버(Server)와 호스팅(Hosting): 전문 식당 임대

    요리를 모든 손님에게 24시간 제공하려면 완벽한 시설을 갖춘 '전문 식당'이 필요합니다. 이 식당이 바로 서버입니다.
    물론 우리 집 주방(로컬 컴퓨터)도 식당처럼 운영할 수는 있어요. 하지만 전기요금이나 보안, 관리 문제로 비효율적입니다. 그래서 전문 식당을 '임대'하기로 합니다. 특정한 공간을 빌리는 거죠. 이 서비스를 호스팅이라고 해요. 임대료와 마찬가지로 대부분의 서버는 호스팅 비용을 내야 이용할 수 있습니다. 하지만 호스팅 비용을 내고 모든 관리를 맡기기 때문에, 우리는 요리(웹사이트 운영)에만 집중할 수 있게 됩니다.

  3. 3

    네트워크 (Network): 식당으로 찾아오는 길

    네트워크(인터넷)는 손님들이 식당으로 찾아올 수 있게 하는 '교통망'입니다. 사용자들은 주소(URL)를 보고, 이 교통망을 통해 우리의 웹사이트(식당)에 방문하게 됩니다.
