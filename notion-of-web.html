1. 웹사이트의 구성 요소

구조 잡기 (HTML)

웹페이지의 뼈대입니다. 제목, 본문, 이미지 등 웹페이지에 들어갈 콘텐츠와 구조를 정의합니다.

스타일 입히기 (CSS)

웹페이지의 디자인입니다. 글자 크기, 색상, 배치(레이아웃) 등을 지정하여 시각적인 표현을 담당합니다.

기능 구현 (JavaScript)

웹페이지의 동작입니다. 버튼 클릭 시 반응하거나 데이터를 처리하는 등 사용자와의 상호작용 기능을 구현합니다.

통합 개발 환경과 도구의 활용

과거에는 HTML, CSS, JavaScript 파일을 각각 분리해서 작업했으나, 최근에는 개발 편의성과 유지보수를 위해 이들을 하나의 컴포넌트(Component) 단위로 통합하여 개발(All-in-one)하는 경우가 많습니다.

이 과정에서 개발자들은 처음부터 모든 코드를 작성하지 않고, 디자인 속도를 획기적으로 높여주는 나, 복잡한 기능을 손쉽게 구현할 수 있도록 도와주는 와 같은 효율적인 도구들을 적극적으로 사용합니다.

* 파란색 글씨를 클릭하면 용어 설명을 볼 수 있습니다.

2. 프론트엔드와 백엔드

프론트엔드 (Frontend)

사용자가 웹사이트에 접속했을 때 눈에 보이는 화면과 인터페이스를 만드는 영역입니다. 사용자의 입력(클릭, 스크롤 등)을 받아 처리합니다.

백엔드 (Backend)

눈에 보이지 않는 서버 시스템과 데이터를 관리하는 영역입니다. 회원 정보를 저장하거나 결제 시스템을 연동하는 등의 뒷단 처리를 담당합니다.

3. 웹사이트가 공개되는 과정

  • 1

    로컬 (Local)

    개발자의 개인 컴퓨터 환경을 '로컬'이라고 부릅니다. 이곳에서는 자유롭게 코드를 작성하고 수정하며, 웹사이트가 의도한 대로 작동하는지 테스트하는 작업을 수행합니다. 하지만 이 공간은 외부 인터넷망과 연결되지 않은 개인적인 공간이기 때문에, 다른 사람들이 내 웹사이트에 접속하여 볼 수는 없습니다.

  • 2

    서버와 호스팅 (Server & Hosting)

    내가 만든 웹사이트를 다른 사람들이 볼 수 있게 하려면, 24시간 내내 인터넷에 연결되어 켜져 있는 고성능 컴퓨터인 '서버'에 파일들을 올려야 합니다. 개인이 이런 서버 컴퓨터를 직접 갖추고 관리하기는 어렵기 때문에, 전문 업체로부터 서버의 일정 공간을 임대하여 사용하는 것을 '호스팅' 서비스라고 합니다.

  • 3

    네트워크 (Network)

    사용자가 웹 브라우저의 주소창에 'www.google.com'과 같은 도메인 주소(URL)를 입력하면, 인터넷이라는 거대한 네트워크망을 통해 해당 주소가 가리키는 서버를 찾아갑니다. 서버는 요청을 확인하고 저장된 웹사이트 파일(HTML, CSS 등)을 사용자의 컴퓨터로 전송하며, 브라우저는 이 코드를 해석하여 우리가 보는 화면으로 보여줍니다.

Tailwind CSS란?

CSS 파일에 스타일 코드를 일일이 작성하는 전통적인 방식 대신, HTML 태그 안에 미리 정의된 스타일 도구(Class)를 적어 넣는 방식의 도구입니다.

예시:

class="text-center bg-blue-500"라고 적기만 하면, 별도의 CSS 파일 없이도 파란 배경에 가운데 정렬된 글씨가 만들어집니다.

JavaScript 라이브러리란?

이미지 슬라이드, 차트, 데이터 처리 등 자주 사용되는 복잡한 기능들을 개발자들이 미리 만들어 둔 '코드 모음집'입니다.

장점:

바퀴를 다시 발명할 필요가 없는 것처럼, 이미 검증된 코드를 가져와 사용하기 때문에 개발 시간을 단축하고 오류를 줄일 수 있습니다.
