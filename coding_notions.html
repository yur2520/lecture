2. 웹 개발의 핵심 원리

1. DOM - 웹의 '뼈대'

DOM(Document Object Model)은 브라우저가 HTML 문서를 이해하도록 만든 **트리(Tree) 구조의 설계도**입니다. 자바스크립트는 DOM을 조작하여 웹 페이지를 변경합니다.

2. JSON - 웹의 '혈액'

JSON(JavaScript Object Notation)은 서버와 클라이언트가 데이터를 주고받을 때 사용하는 **표준 '데이터 택배 상자'**입니다. 사람이 읽기 쉽고 기계가 분석하기도 편해 사실상 표준으로 사용됩니다.

3. 가상 DOM - '효율적인 일꾼'

리액트는 **가상 DOM(진짜 DOM의 복사본)**을 사용합니다. 변경분을 가상 DOM에서 먼저 계산한 뒤, 최소한의 변경분만 찾아 진짜 DOM에 한 번에 적용하여 성능을 향상시킵니다.

3-2. AI 앱 빌더와 노코드: 'AI 건축가'에게 맡기기

위의 클라우드 IDE들이 '내가 직접 코딩하는 장소'를 제공한다면, 아래 서비스들은 개발의 패러다임을 바꿉니다.

러버블 (Lovable)

AI가 코드를 대신 짜주는 'AI 건축가'입니다. 자연어로 앱을 설명하면, Supabase, Vercel 등 최고의 도구들을 조합하여 실제 React 소스 코드를 생성하고 배포까지 해줍니다. 결과물이 실제 코드라 자유로운 수정과 확장이 가능합니다.

버블 (Bubble)

코드가 전혀 없는 '올인원 노코드 빌더'입니다. 드래그 앤 드롭 방식으로 UI를 만들고, 워크플로우를 설정하여 앱을 완성합니다. 비개발자가 가장 빠르게 아이디어를 실제 작동하는 앱으로 만들 수 있는 강력한 도구입니다.

4. 웹 개발의 전체 구조 이해하기

프론트엔드와 백엔드의 역할 구분

  • 프론트엔드 (Frontend): 사용자의 기기(웹 브라우저, 스마트폰)에서 직접 실행되는 영역입니다. 사용자가 보는 화면(UI)과 상호작용(UX)을 모두 책임집니다.
  • 백엔드 (Backend): 서버 컴퓨터에서 실행되는, 눈에 보이지 않는 영역입니다. 데이터 처리 및 저장, 사용자 인증 등 서비스의 핵심 로직을 담당합니다.

서버 관련 핵심 용어 정리

  • 호스팅 (Hosting): 개발자가 만든 코드를 인터넷 사용자가 접속할 수 있도록 서버 컴퓨터의 공간을 임대해주는 서비스입니다.
  • API (Application Programming Interface): 프로그램 간의 약속된 통신 규칙입니다. 프론트엔드는 이 규칙에 따라 백엔드에 데이터를 요청하고 응답받습니다.
  • BaaS (Backend as a Service): 데이터베이스, 사용자 인증 등 자주 사용되는 백엔드 기능들을 미리 구현하여 API 형태로 제공하는 서비스입니다. (예: Firebase, Supabase)

클라우드 서비스 이해하기: 백화점 비유 (IaaS, PaaS, BaaS)

AWS, GCP 같은 클라우드 서비스는 모든 것을 제공하는 거대한 '백화점'입니다.

IaaS (Infrastructure...)

'텅 빈 상가' 임대. (예: GCP Compute Engine)
가장 기본적인 공간(서버)만 빌리는 것. 인테리어, 주방 설비 등 모든 것을 직접 해야 합니다. 자유도가 높지만 가장 복잡합니다.

PaaS (Platform...)

'모든 설비가 갖춰진 주방' 임대. (예: Render, Heroku)
주방 설비는 신경 쓸 필요 없이, 내 요리사(백엔드 코드)만 데려와서 바로 요리를 시작할 수 있습니다.

BaaS (Backend...)

'백화점 내 전문 식당' 이용. (예: Firebase)
주방을 빌릴 필요도 없이, 그냥 식당에 가서 완성된 요리를 주문해 먹는 것과 같습니다. 가장 편리합니다.

5. 결론: 무엇을 선택해야 할까?

어떤 기술을 선택할지는 개발자의 경험과 프로젝트의 목표에 따라 달라집니다.

  • 웹 개발 경험이 풍부하다면?
    자바스크립트와 리액트에 익숙하다면, 리액트 네이티브를 통해 모바일 개발로 쉽게 확장할 수 있습니다.
  • 완벽한 UI 일관성이 중요하다면?
    모든 플랫폼에서 픽셀 단위까지 동일한 디자인과 경험을 제공하고 싶다면 플러터가 더 나은 선택입니다.
  • 하나의 코드로 모든 플랫폼을 지원하고 싶다면?
    모바일, 웹, 데스크톱 앱을 모두 만들고 싶다면 플러터가 가장 강력한 솔루션입니다.
