1. 기본 원칙

  • 1. 명확한 위치 지정

    수정하고 싶은 정확한 영역을 알려주세요.

    예: "화면 상단의 헤더 부분", "가입 버튼"

  • 2. 구체적인 방향 제시

    '보기 좋게' 보다는 '가운데 정렬', '3열 그리드'처럼 구체적으로 지시하세요.

  • 3. 정확한 색상과 수치

    '파란색' 대신 '진한 파란색(#2563EB)', '넓게' 대신 '30px'를 사용하세요.

2. 레이아웃 명칭

웹 페이지의 구역(Zone)

헤더 (Header): 웹사이트 최상단. 로고, 메인 메뉴(GNB)가 위치합니다.

메인 콘텐츠 (Main Content): 페이지의 핵심 정보가 담기는 중앙 영역입니다.

사이드바 (Sidebar): 좌/우측에 위치하여 보조 메뉴나 광고, 필터 등을 표시합니다.

푸터 (Footer): 최하단. 저작권, 회사 정보, 이용 약관 등이 위치합니다.

내비게이션 (Navigation): 다른 페이지로 이동하기 위한 링크 모음(메뉴바)입니다.

배치 및 정렬 (Arrangement)

그리드 (Grid): 바둑판처럼 행과 열을 맞추어 요소를 배치하는 방식입니다.

플렉스 (Flexbox): 아이템들을 가로 또는 세로 한 방향으로 유연하게 배치합니다.

정렬 (Alignment): "중앙 정렬(Center)", "양쪽 맞춤(Justify)" 등으로 표현합니다.

여백 (Margin): 요소의 '바깥쪽' 공간. 요소끼리 떨어뜨릴 때 사용합니다.

안쪽 여백 (Padding): 요소의 '안쪽' 공간. 테두리와 내용 사이의 간격입니다.

간격 (Gap): 그리드나 플렉스 아이템 사이의 거리를 의미합니다.

4. 색상과 폰트(Typography)

색상 지정하기

HEX 코드

#2563EB 처럼 # 뒤에 6자리 코드로 색을 지정합니다.

Tailwind CSS 컬러

blue-500, sky-200 처럼 미리 정의된 이름을 사용합니다.

구글 폰트 (Google Fonts)

웹에서 무료로 사용할 수 있는 다양한 폰트를 적용하는 방법입니다.

  1. fonts.google.com 접속
  2. 원하는 폰트를 검색하고 선택 (예: Noto Sans KR)
  3. 'Get font' > 'Embed code' 클릭
  4. 제공된 <link> 코드를 복사
  5. 프롬프트에 "Noto Sans KR 폰트를 CDN으로 추가해 줘"라고 요청
/* CSS 예시 */
font-family: 'Noto Sans KR', sans-serif;

6. 기타 고려사항

반응형 & 적응형 디자인

반응형 디자인 (Responsive): 화면 크기에 따라 요소가 유동적으로 늘어나거나 줄어듭니다.

적응형 디자인 (Adaptive): 모바일(스마트폰), 태블릿, 데스크탑 등 특정 크기 지점(Breakpoint)마다 레이아웃을 다르게 보여줍니다.

프롬프트 팁:

"모바일 화면(768px 이하)에서는 메뉴를 숨기고 햄버거 아이콘 버튼을 보여줘."
