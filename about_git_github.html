Git & GitHub 핵심 가이드

개발자의 필수 도구, Git과 GitHub

코드의 변경 이력을 관리하고, 다른 사람들과 공유 & 협업하기

Git

버전 관리 시스템

Git은 파일의 변경 사항을 시간 순서대로 기록하고 관리하는 프로그램입니다. 오픈 소스 프로그램으로 누구나 무료로 사용할 수 있으며, 게임의 세이브 포인트처럼 코드의 특정 상태를 저장(Commit)해두고 언제든지 원하는 시점으로 되돌아갈 수 있게 해 줍니다. 개발 버전을 제어하는 다른 프로그램들도 있으나, 현재는 Git이 가장 널리 쓰이고 있으며 그 외의 시스템은 특정 환경이나 용도에 따라 제한적으로 사용되고 있습니다.

주요 특징

  • 분산 버전 관리 (오프라인 작업 가능) : 완전히 독립적인 로컬 환경에서 작업하고 각 로컬 환경마다 버전을 관리할 수 있도록 해 줍니다.
  • 빠르고 효율적인 성능 : 거의 모든 작업을 로컬에서 수행하고, 메인 브랜치에 내보내기를 수행할 때에만 서버와 통신하므로 개발 시 쾌적한 속도를 경험할 수 있습니다.
  • 브랜치(Branch) 기능 : 나뭇가지가 뻗어나가는 것과 같이 각 분기점을 만들어 작업하고, 나중에 병합시킬 수 있습니다.

핵심 개념

  • Repository (저장소): 프로젝트 저장소
  • Commit (커밋): 변경 사항을 저장하는 단위
  • Branch (브랜치): 독립적인 작업 흐름
  • Merge (병합/머지): 브랜치를 하나로 병합

GitHub

Git 호스팅과 협업 공간을 제공하는 플랫폼

GitHub는 Git으로 관리되는 프로젝트를 저장하고 공유하는 웹사이트입니다. 내 컴퓨터에 있는 Git 저장소를 GitHub에 올려두면 안전하게 백업할 수 있고, 다른 사람들과 함께 코드를 보며 협업할 수 있습니다.

주요 기능

  • 원격 저장소(Remote Repository) 제공
  • 코드 리뷰 (Pull Request)
  • 이슈 트래킹 및 프로젝트 관리
  • 정적 웹사이트 호스팅 (GitHub Pages)

협업의 핵심

  • Push: 내 컴퓨터의 변경 이력을 GitHub로 업로드
  • Pull: GitHub의 변경 이력을 내 컴퓨터로 다운로드
  • Pull Request: 내 코드를 원본 프로젝트에 반영 요청
구체적인 사용 방법 및 예시

GitHub 외의 다른 Git 호스팅 플랫폼

GitLab

하나의 애플리케이션으로 전체 DevOps(개발-운영) 라이프사이클을 관리하는 것을 목표로 합니다.

  • 강력한 CI/CD 파이프라인 기능 내장
  • 설치형(Self-hosted) 버전 제공으로 보안 강화

Bitbucket

Atlassian 제품군(Jira, Confluence 등)과의 뛰어난 연동성을 자랑합니다.

  • Jira와의 긴밀한 통합으로 이슈 추적 용이
  • 프라이빗 저장소를 소규모 팀에게 무료로 제공

GitHub를 많이 사용하는 이유

다양한 플랫폼이 있음에도 GitHub가 압도적인 점유율을 보이는 데에는 여러 이유가 있습니다. 가장 먼저 시장을 선점하여 수많은 개발자와 오픈소스 프로젝트가 모이는 거대한 생태계를 구축했습니다. 특히 공개 저장소(Public Repository)를 무료로 제공하여 전 세계 오픈소스 프로젝트가 다수 모이게 된 점이 결정적이었습니다. 또한 직관적이고 사용하기 쉬운 UI/UX를 제공하며, 코드 리뷰 및 협업을 위한 풀 리퀘스트(Pull Request) 시스템을 대중화시켰습니다. GitHub Actions(CI/CD), GitHub Pages(웹 호스팅) 등 개발에 필요한 다양한 부가 기능을 통합하여 개발자에게 '올인원(All-in-one)' 플랫폼 경험을 제공하는 것도 큰 장점입니다.

내 컴퓨터에서는 Git으로 체계적으로 코드 변경 이력을 관리하고, 그 결과를 GitHub와 같은 호스팅 플랫폼에 올려 안전하게 백업하고 동료들과 효율적으로 협업하는 것이 일반적인 개발 방식입니다.

이 페이지는 개발 학습을 돕기 위해 만들어졌습니다.
