Git & GitHub 워크플로우 가이드

Git을 사용하여 프로젝트 버전을 관리하고 GitHub와 연동하는 전체적인 과정을 안내합니다.

1. 사전 준비: Git 설치

가장 먼저, 컴퓨터에 Git이 설치되어 있는지 확인하고, 없다면 설치해야 합니다.

  • 설치 확인: 터미널(macOS/Linux)이나 명령 프롬프트(Windows)에서 git --version을 입력하여 버전을 확인합니다.
  • 설치: 버전이 표시되지 않으면 아래 링크에서 다운로드하여 설치하세요.
    • Git for Windows
    • Git for macOS (Homebrew 사용 시: brew install git)
    • Git for Linux (Debian/Ubuntu: sudo apt-get install git)

2. 프로젝트 시작하기: 두 가지 방법

프로젝트를 시작하는 방법은 크게 두 가지입니다. 내 컴퓨터에서 새로 시작하거나, GitHub에 있는 것을 가져올 수 있습니다.

방법 A: 내 컴퓨터에서 새 프로젝트 시작하기

로컬 폴더를 Git 저장소로 만들고 GitHub에 연결합니다.

  1. 프로젝트 폴더로 이동

    터미널을 열고 프로젝트의 루트 폴더로 이동합니다. (your_project는 실제 폴더 이름으로 변경)

    cd path/to/your/project
  2. Git 저장소로 초기화

    아직 Git 저장소가 아니라면, 다음 명령어로 초기화합니다. 폴더에 .git이라는 숨김 폴더가 생성됩니다.

    git init
  3. 원격 저장소(GitHub) 연결

    GitHub에 새 저장소(repository)를 만든 후, 아래 명령어로 로컬 저장소와 연결합니다. URL은 GitHub 저장소 페이지에서 복사할 수 있습니다.

    # HTTPS 방식
git remote add origin https://github.com/your-username/your-repo-name.git

방법 B: GitHub 프로젝트를 내 컴퓨터로 복제하기

GitHub에 이미 존재하는 프로젝트를 내 컴퓨터로 그대로 복제해옵니다.

이 방법은 가장 간단하고 일반적입니다. git clone 명령어 하나로 저장소 복제, 폴더 생성, 원격 저장소 연결이 모두 완료됩니다.

  1. 저장소 URL 복사

    복제하고 싶은 GitHub 저장소 페이지로 이동하여 'Code' 버튼을 누르고 HTTPS URL을 복사합니다.

  2. 프로젝트 복제(Clone)

    터미널에서 프로젝트를 저장하고 싶은 상위 폴더로 이동한 후, 다음 명령어를 실행합니다.

    git clone https://github.com/your-username/your-repo-name.git

    이제 your-repo-name 폴더가 생성되었을 겁니다. 해당 폴더로 이동하면 바로 작업을 시작할 수 있습니다.

    cd your-repo-name

3. 작업 흐름: 수정, 저장, 그리고 공유

프로젝트를 시작했다면, 이제 코드를 수정하고 그 변경사항을 GitHub에 올리는 과정을 반복하게 됩니다.

  1. 변경된 파일 스테이징(Staging)

    작업한 파일들을 '커밋(commit)'이라는 기록으로 남기기 위해 준비 단계인 '스테이징 영역'에 추가합니다.

    모든 변경사항을 한 번에 추가하려면:

    git add .

    특정 파일만 추가하려면:

    git add path/to/your/file.dart
  2. 변경 사항 커밋(Commit)

    스테이징된 파일들을 하나의 의미 있는 작업 단위로 묶어 로컬 저장소에 기록을 남깁니다. 커밋 메시지는 '무엇을 왜' 변경했는지 명확하게 작성하는 것이 중요합니다.

    git commit -m "feat: 시간표 관리 기능 추가"
  3. GitHub에 푸시(Push)

    로컬 저장소에 쌓인 커밋들을 원격 저장소(GitHub)에 업로드하여 다른 사람들과 공유하고 백업합니다.

    # 보통 기본 브랜치 이름은 'main' 입니다.
git push origin main

    *만약 처음 푸시하는 경우 -u 옵션을 붙여 git push -u origin main으로 실행하면, 다음부터는 git push만 입력해도 됩니다.

💡 전문가 팁: 브랜치(Branch) 활용

중요한 기능을 추가하거나 코드를 크게 변경할 때는 main 브랜치에서 직접 작업하는 대신, 새로운 브랜치를 만들어 독립적으로 작업하는 것이 안전하고 효율적입니다.

  1. 새 브랜치 생성 및 이동: feature/new-login 이라는 이름의 브랜치를 만들고 그 브랜치로 이동합니다. 
    git checkout -b feature/new-login
  2. 작업 및 커밋: 위 3번 과정(add, commit)을 동일하게 진행합니다.
  3. 새 브랜치 푸시: 작업한 브랜치를 GitHub에 올립니다. 
    git push origin feature/new-login
  4. Pull Request (PR) 생성: GitHub에서 feature/new-login 브랜치를 main 브랜치로 병합(merge)해달라는 요청(Pull Request)을 생성합니다. 동료의 코드 리뷰 후 안전하게 메인 코드에 반영할 수 있습니다.

이 가이드를 따라 Git과 GitHub을 사용하면 프로젝트를 체계적으로 관리할 수 있습니다.
작업 시작 전에는 항상 git pull로 최신 코드를 받아오는 습관을 들이는 것이 좋습니다!
