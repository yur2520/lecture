유용하게 쓸 수 있는 웹 디자인 관련 키워드

웹 페이지의 구역 명칭

헤더 (Header): 웹사이트의 가장 윗부분. 보통 로고나 메뉴가 들어갑니다.

푸터 (Footer): 웹사이트의 가장 아랫부분. 회사 정보나 저작권 표시가 들어갑니다.

사이드바 (Sidebar): 페이지의 왼쪽이나 오른쪽에 세로로 길게 위치한 메뉴 또는 광고 영역입니다.

메인 콘텐츠 (Main Content): 페이지의 가장 중요한 실제 내용이 표시되는 중앙 영역입니다.

내비게이션 레일 & 바 (Navigation Rail & Bar): 화면의 옆(레일)이나 위(바)에 위치하여 다른 페이지로 이동할 수 있게 도와주는 메뉴입니다.

내용의 배치 (레이아웃)

그리드 (Grid): 페이지를 표(격자)로 나눠 질서정연하게 배치하는 방식입니다.

플렉스박스 (Flexbox): 한 줄 또는 한 열 안의 내용물들을 유연하게 정렬하는 방식입니다.

정렬 (Alignment): `가운데 정렬`, `왼쪽/오른쪽 정렬` 등 내용물을 특정 위치로 붙이는 것을 말합니다.

간격 (Gap) / 여백 (Margin): 요소와 요소 사이의 거리를 의미합니다.

안쪽 여백 (Padding): 테두리 '안쪽'의 여백. 내용물과 테두리 사이의 공간을 만들 때 사용합니다.

캐러셀 (Carousel): 여러 이미지나 콘텐츠를 옆으로 돌려가며 볼 수 있는 슬라이드입니다.

체크박스/라디오/토글: 사용자가 선택할 수 있는 다양한 모양의 버튼들입니다.

카드 (Card): 이미지, 제목, 설명 등 관련 정보를 하나의 사각형 안에 묶어놓은 형태입니다. 지금 이 설명을 담고 있는 흰색 사각형도 하나의 카드 레이아웃입니다.

색상 표현법

HEX 코드: #2563EB 처럼 #과 6자리 코드로 특정 색상을 정확하게 가리키는 고유 번호입니다.

Tailwind CSS 색상: blue-600 처럼 Tailwind CSS에 미리 지정된 색상 이름을 사용하는 방식입니다. 영어 단어는 색상, 숫자는 진한 정도를 나타냅니다.

반응형 디자인

반응형 디자인 (Responsive Design): 웹사이트를 보는 기기(컴퓨터, 스마트폰 등)의 화면 크기에 맞춰 디자인이 자동으로 최적화되는 방식입니다.

상황별 프롬프트 작성 예시

상황 1

페이지 구조 변경

일반적인 요청

"옆에 메뉴가 있었으면 좋겠어."

구체적인 요청

"페이지 왼쪽에 사이드바를 만들고, 오른쪽에는 메인 콘텐츠 영역을 만들어 줘."

상황 2

요소 색상 변경

일반적인 요청

"버튼 색깔 좀 바꿔줘."

구체적인 요청

"'제출' 버튼의 배경색은 HEX 코드 `#60a2c4`로, 글자색은 흰색으로 변경해 줘."

상황 3

항목 정렬 및 간격 조정

일반적인 요청

"자료 목록이 너무 정신없어 보여."

구체적인 요청

"자료 목록을 3열 그리드 레이아웃으로 만들고, 각 항목의 간격은 16px로 설정해 줘."

상황 4

반응형으로 디자인

일반적인 요청

"모바일에서도 잘 보이게 해줘."

구체적인 요청

"화면 너비가 768px 이하인 모바일 환경에서는 자료 목록을 3열이 아닌 1열로 보여줘."
